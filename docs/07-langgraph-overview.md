# 07 为什么要看 LangGraph

原文来源：

- LangGraph Overview
- 链接：https://docs.langchain.com/oss/python/langgraph/overview

为什么今天读这个：

当你开始从“单次工具调用”走向“多步可控流程”，就会需要图结构、状态和节点编排视角。

中文精读：

LangGraph 可以先别把它看成某个必须掌握的框架，而要把它看成一种工程组织方式：

1. 用节点表示动作
2. 用边表示流转
3. 用状态表示上下文和中间结果

它适合的不是最简单的聊天，而是：

1. 有多步流程
2. 有条件分支
3. 有状态延续
4. 需要可控和可观测

对你来说，LangGraph 的价值主要有两层：

1. 学习层
- 帮你把 Agent 逻辑画清楚，不再是一坨 Prompt

2. 工程层
- 帮你理解为什么 Agent 编排最终会落到状态机、图结构、节点契约

今天先不用深学 API，先抓住思想：

1. Agent 不是只有“模型调用”
2. 它本质上还是有状态的流程系统
3. 只不过其中部分决策由模型完成

你只需要记住的 3 个点：

1. LangGraph 强调“状态 + 节点 + 边”的编排思维。
2. 它适合多步、可恢复、可观测的 Agent 流程。
3. 学 LangGraph 的重点不是记 API，而是学状态化工作流思维。

和你的转型关系：

这很适合你，因为你已经有系统流程和集成经验。你本来就擅长“节点怎么串、异常怎么回退”，LangGraph 只是让这件事进入 Agent 世界。

# 08 为什么 Python 第一站建议是 FastAPI

原文来源：

- FastAPI Tutorial
- 链接：https://fastapi.tiangolo.com/tutorial/

为什么今天读这个：

你补 Python 的目标不是刷语法题，而是尽快拥有一个能承载 Agent 服务的工程壳。

中文精读：

FastAPI 很适合你当前阶段，原因不是“它最火”，而是它刚好满足你转型前期的几个条件：

1. 上手快
- 很快就能写出接口

2. 工程感强
- 请求模型、响应模型、校验、文档、错误处理都比较完整

3. 和 Agent 服务天然匹配
- 你后面做 task API、tool API、trace API、eval API 都用得上

FastAPI 教程本身有一个很大的优点：

1. 结构循序渐进
2. 代码块可以直接运行
3. 既能入门，也能当查阅手册

对你现在最重要的，不是一下子学完整本教程，而是先建立这个意识：

1. Python 在你这里不是脚本语言
2. 它要成为一个能写服务、写测试、写配置、写日志的工程语言

所以前期你看 FastAPI，真正要关注的是：

1. 路由怎么定义
2. 请求和响应模型怎么组织
3. 异常怎么处理
4. 项目结构怎么拆
5. 怎么配测试

你只需要记住的 3 个点：

1. Python 补齐要尽快落到服务工程，而不是停留在语法。
2. FastAPI 是你做最小 Agent 服务最合适的第一站。
3. 后面所有 tool calling、trace、eval 都可以挂在这个服务壳上。

和你的转型关系：

你本来就是后端工程师，所以 Python 学习不应该走“零基础入门”路线，而应该直接走“最小可交付服务”路线。

今晚可动手：

写出你理想中的最小 FastAPI Agent 服务接口清单，例如：

- `GET /health`
- `POST /task`
- `GET /traces/{id}`

# 阅读地图

这批材料不是按“框架名”排的，而是按你转型时最该先建立的概念顺序排的。

建议顺序：

1. `01-agent-vs-workflow.md`
目的：先把 “什么叫 Agent” 讲明白，避免后面一切都混在一起。

2. `02-function-calling.md`
目的：理解模型为什么能“调工具”，以及系统边界在哪里。

3. `03-mcp-overview.md`
目的：理解 MCP 到底解决什么问题，它和普通函数调用有什么差别。

4. `04-openai-agents-sdk.md`
目的：把 Agent 放回真正的工程对象里看：状态、工具、handoff、guardrail、trace。

5. `05-rag-paper.md`
目的：建立 RAG 的第一性原理，别只会说“向量库”。

6. `06-react-paper.md`
目的：理解边思考边行动为什么成立，以及它的优缺点。

7. `07-langgraph-overview.md`
目的：开始建立工作流编排视角。

8. `08-fastapi-tutorial.md`
目的：把阅读转到代码承载层，为 Python 工程化铺路。

阅读原则：

1. 每篇先看 `为什么今天读这个`
2. 只记住 `3 个点`
3. 先会说，再会写，再会做

补充目录：

- `references/user-roadmaps/00-index.md`
作用：回看你自己的路线、JD 拆解、面试题和月计划。

# Agent面试题目（高频梳理+参考答案）

更新时间：2026-04-26
定位：面向 Agent 全栈开发工程师（偏字节/阿里/DeepSeek同类岗位）
使用方式：每题先用 30-60 秒口述“结论+方案+取舍”，再展开细节。

## 一、Agent基础与架构

1. 什么是 Agent？和普通 LLM 调用有什么区别？
- 参考答案：普通 LLM 调用通常是“单轮输入-输出”；Agent 是“感知-规划-执行-反馈-记忆”的闭环系统，核心在于可调用工具、可维护状态、可多步完成目标任务。

2. 单 Agent 和多 Agent 怎么选？
- 参考答案：任务边界清晰、步骤短、跨角色协作少时用单 Agent；任务复杂、角色分工明显、需要并行与解耦时用多 Agent。多 Agent 复杂度更高，必须有路由、通信协议和失败补偿机制。

3. Agent 常见组件有哪些？
- 参考答案：Planner（规划）、Executor（执行）、Tool Router（工具路由）、Memory（短/长期记忆）、State Store（状态存储）、Evaluator（评测与反思）、Guardrail（安全约束）。

4. ReAct 和 Plan-and-Execute 的区别？
- 参考答案：ReAct是边想边做，响应快但可能反复；Plan-and-Execute 先全局规划再执行，更稳定可控，适合复杂任务。生产中常用混合策略：粗规划 + 局部 ReAct。

5. 如何避免 Agent 死循环？
- 参考答案：设置最大步数、重复动作检测、无收益迭代熔断、失败重试上限、目标完成判定器；并记录每步结构化轨迹用于回放排障。

## 二、Memory 与上下文工程

6. 短期记忆和长期记忆有什么区别？
- 参考答案：短期记忆服务当前会话任务（高时效）；长期记忆保存跨会话偏好/事实（高复用）。短期常在窗口内摘要，长期通常存储在 DB/向量库。

7. 记忆检索应该按 user_id 还是 session_id？
- 参考答案：不能单键。session_id 适合短期过程恢复，user_id 适合长期偏好，还应叠加 task_id/workspace_id/entity_id 做多维过滤，避免记忆污染。

8. 上下文工程解决什么问题？
- 参考答案：不是只解决“token 不够”，而是解决“给模型最小但充分的信息集”。核心是信息选择、压缩、排序和阶段性投影。

9. 什么时候触发上下文压缩？
- 参考答案：token 阈值逼近、任务阶段切换、工具返回激增、会话轮次过长。推荐“硬阈值 + 事件触发”双机制。

10. 为什么通常优先压缩模型输出而非用户输入？
- 参考答案：模型输出往往冗余更高；用户输入通常携带目标和约束。先压输出可降低失真风险，但超长用户输入也要结构化抽取。

## 三、RAG 与检索

11. 向量检索和关键词检索怎么选？
- 参考答案：关键词检索精确、可解释，适合术语/ID匹配；向量检索语义召回强，适合自然语言问题。线上一般做混合检索 + 重排。

12. RAG 常见链路是什么？
- 参考答案：Query改写 -> 召回（BM25/向量）-> 重排 -> 上下文构造 -> 生成 -> 引用与置信度输出。

13. 分块（chunk）怎么做更稳？
- 参考答案：按语义边界分块，控制块大小和重叠；对于结构化文档保留标题层级与元数据，便于过滤和追溯。

14. 如何降低“召回到了但没用上”？
- 参考答案：做答案约束模板、引用强制、证据优先排序、无证据拒答策略，并在评测中单独统计“evidence usage rate”。

15. RAG 评估指标有哪些？
- 参考答案：召回命中率、MRR/NDCG、答案正确率、引用一致性、幻觉率、平均延迟、单请求成本。

## 四、Tool Calling / MCP / Function Calling

16. Function Calling 和 MCP 的区别？
- 参考答案：Function Calling 多是模型侧的函数参数化调用规范；MCP 更偏“统一工具接入与上下文交换协议”，关注工具发现、权限、会话上下文与生态互联。

17. Tool Router 怎么设计？
- 参考答案：规则路由 + 模型路由结合。先做硬规则过滤（权限/参数/领域），再让模型在候选工具中选择，最后做结果校验与回退。

18. 工具调用失败怎么处理？
- 参考答案：分类处理（超时/鉴权/参数/业务失败），幂等重试、退化策略、人工兜底；同时向上层返回可观察错误码与可解释提示。

19. 多工具并发调用如何控风险？
- 参考答案：并发配额、超时隔离、熔断限流、写操作串行化、补偿事务；避免多个 Agent 同时写同一资源导致冲突。

20. 什么时候用 Skill，什么时候直接调 API？
- 参考答案：可复用流程和策略封装成 Skill；一次性能力或简单调用直接 API。Skill 的价值是复用、治理、评测和可观察。

## 五、工程化与平台能力

21. 如何设计 Agent 评测平台？
- 参考答案：三层：数据集层（任务集/标注）、执行层（批量运行/回放）、分析层（成功率/延迟/成本/错误分布）。支持版本对比与回归报警。

22. 轨迹回放工具要看什么？
- 参考答案：每步思考摘要、工具入参出参、状态快照、token/cost、失败原因、重试链路；目标是快速定位“哪一步开始偏航”。

23. 如何做线上稳定性保障？
- 参考答案：超时、重试、熔断、降级、缓存、队列削峰、灰度发布；同时做SLO（成功率、P95延迟、成本上限）治理。

24. 为什么 Agent 系统比普通后端更需要可观测性？
- 参考答案：其链路长、非确定性高、外部依赖多。没有 step 级日志和追踪，很难定位失败根因与性能瓶颈。

25. CI/CD 在 Agent 项目里怎么做？
- 参考答案：代码测试 + Prompt回归测试 + 评测集回放 + 安全扫描 + 镜像构建发布；新增“模型/Prompt 变更门禁”。

## 六、容器与部署（岗位高频）

26. 为什么要求 Docker/K8s？
- 参考答案：为了解决环境一致性、弹性扩缩容、隔离与灰度发布。Agent 服务常依赖多组件，容器化是生产化基础。

27. Agent 服务在 K8s 上要关注哪些点？
- 参考答案：资源配额、HPA策略、探针健康检查、配置与密钥管理、日志采集、节点亲和、冷启动时延。

28. 如何控制推理成本？
- 参考答案：模型分级路由、小模型前置、缓存、上下文压缩、并发治理、批处理、失败快速返回。

## 七、AI Coding 工具（近年高频）

29. 你怎么使用 Claude Code/Cursor/Copilot？
- 参考答案：把它们当结对工程师：需求拆解->骨架生成->测试补齐->review->重构；关键代码和架构决策必须人工把关。

30. 如何避免 AI 代码“能跑但不可维护”？
- 参考答案：强制测试、静态检查、代码审查、设计文档、无AI手写训练；不合并自己解释不清的代码。

31. 上下文过长导致 AI 编码效果下降怎么办？
- 参考答案：任务切片、上下文最小化、模块边界清晰、只提供必要文件、分阶段提示词；避免一次性大而全请求。

## 八、后端基础（面试必问）

32. 为什么 Agent 岗还会问 MySQL/Redis/MQ/网络？
- 参考答案：Agent 最终要落地在线系统，本质还是分布式后端问题：一致性、性能、可靠性、可用性。

33. Redis 持久化和高可用怎么回答？
- 参考答案：RDB/AOF 各自取舍，结合主从+哨兵或集群；业务上要有缓存穿透/击穿/雪崩防护。

34. 慢查询怎么排查？
- 参考答案：先看执行计划和索引命中，再看SQL写法与数据分布，必要时分库分表/归档冷数据，最后加监控和告警闭环。

35. 并发场景如何做线程安全？
- 参考答案：先减少共享状态，再用锁/乐观并发/消息串行化。跨服务写冲突用幂等键、去重表、补偿事务。

## 九、系统设计开放题（常见）

36. 设计一个“会议安排 Agent”你会怎么做？
- 参考答案：拆成意图识别、日历检索、冲突检测、候选时间生成、确认与落库、通知发送；加权限校验与失败补偿。

37. 设计一个“高并发优惠券发放 Agent 系统”要点？
- 参考答案：人群分层策略、库存原子扣减、幂等防重、异步发放、失败补偿、审计日志、压测与容量预估。

38. 你如何判断一个 Agent 项目是否成功？
- 参考答案：业务指标（完成率、转化率、人工替代率）+ 工程指标（成功率、延迟、成本、回归稳定性）+ 体验指标（满意度/投诉率）。

## 十、行为与项目深挖（高通过率关键）

39. 面试官为什么反复“拷打项目细节”？
- 参考答案：验证项目真实性、问题定位能力、工程取舍能力。回答要围绕“场景-方案-数据-问题-优化-结果”。

40. 如果线上结果突然变差，你怎么处理？
- 参考答案：先止血（降级/回滚），再分层排查（数据->检索->Prompt->工具->模型->系统），最后复盘并补监控门禁。

41. 你怎么解释一次失败的 Agent 迭代？
- 参考答案：明确失败假设、证据、修复动作和后续防复发机制。面试中要体现“可复盘、可学习、可改进”。

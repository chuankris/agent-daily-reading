# Agent岗位JD技能汇总

更新时间：2026-04-26
适用范围：DeepSeek、字节、阿里及同类 Agent 全栈 / Agent 平台 / Agent Infra 岗位
目的：把岗位要求从“看起来很多”整理成可学习、可验证、可交付的技能地图。

## 1. 核心结论

如果想比较稳地冲击 Agent 全栈开发工程师岗位，能力要求可以归纳为 6 条主线：

1. 语言与编码能力
2. Agent 基础原理与协议能力
3. 前后端工程化能力
4. 平台与基础设施能力
5. AI 编程工具深度使用能力
6. 项目表达与工程协作能力

真正决定面试通过率的，不只是“知道这些词”，而是：
- 能不能做出一个可运行的 Agent 项目
- 能不能解释清楚架构和取舍
- 能不能把系统做得稳定、可观测、可扩展

## 2. 必须掌握的技能

### 2.1 编程语言

1. Python
- 必须程度：高
- 作用：Agent 后端、评测脚本、工具调用、服务编排、数据处理
- 至少达到：能独立写 FastAPI 服务、封装模型调用、写 pytest、做日志和配置管理

2. TypeScript / JavaScript
- 必须程度：高
- 作用：前端页面、轨迹可视化、BFF/API 层、工具控制台
- 至少达到：能独立完成一个简单前端页面和一个 Node/TS 服务

3. Rust
- 必须程度：中高
- 作用：高性能模块、工具运行时、底层服务、稳定性要求高的组件
- 至少达到：能写小型服务或模块，理解所有权、并发、错误处理
- 现实建议：不用一开始学得很深，但面试前最好有真实项目证明

### 2.2 Agent 原理与能力

1. Prompt / Context Engineering
- 必须程度：高
- 至少达到：理解上下文裁剪、信息组织、角色提示、输出约束、失败重试

2. Tool Calling / Function Calling
- 必须程度：高
- 至少达到：能设计工具 schema、处理参数校验、错误回退和结果注入

3. MCP（Model Context Protocol）
- 必须程度：中高
- 至少达到：理解它解决的问题、基本交互方式、和普通函数调用的区别

4. Memory / Planning / Multi-step Agent
- 必须程度：中高
- 至少达到：理解短期记忆、长期记忆、Planner、Executor 的职责拆分

5. RAG 基础
- 必须程度：中
- 至少达到：理解召回、重排、上下文构造、引用、幻觉控制

### 2.3 前后端工程能力

1. 后端服务开发
- 必须程度：高
- 技能点：REST API、鉴权、异常处理、日志、超时、重试、缓存、任务队列

2. 前端开发
- 必须程度：高
- 技能点：React 或同类框架、TS 类型系统、接口联调、空态/异常态处理、列表与详情页

3. 测试能力
- 必须程度：高
- 技能点：单元测试、集成测试、接口测试、回归测试

4. 代码质量
- 必须程度：高
- 技能点：lint、format、代码审查、清晰目录结构、配置管理

## 3. 平台与基础设施能力

### 3.1 容器与部署

1. Docker
- 必须程度：高
- 至少达到：能写 Dockerfile、管理依赖、构建镜像、理解环境隔离

2. Kubernetes
- 必须程度：中高
- 至少达到：理解 Deployment、Service、ConfigMap、Secret、探针、扩缩容

3. Linux 基础
- 必须程度：中高
- 至少达到：会排日志、查进程、看端口、做基本环境排查

### 3.2 工程平台化

1. CI/CD
- 必须程度：高
- 至少达到：能配置基础流水线，完成测试、构建、部署或镜像发布

2. 可观测性
- 必须程度：高
- 至少达到：有结构化日志、错误码、链路跟踪意识，知道如何做告警与回放

3. 稳定性治理
- 必须程度：高
- 至少达到：理解超时、重试、熔断、限流、降级、幂等

## 4. AI 编程工具能力

这是近一年非常关键的一项，很多 JD 已经明确写出来了。

1. Claude Code / Codex / Cursor / Copilot 的使用
- 必须程度：中高
- 至少达到：
  - 会做需求拆解
  - 会用 AI 生成代码骨架
  - 会让 AI 做 review 和测试补齐
  - 会控制上下文，避免 AI 输出失控

2. Prompt 编写能力
- 必须程度：高
- 至少达到：能写结构清晰、约束明确、适合工程任务的 Prompt

3. AI 工具二次开发
- 加分程度：中高
- 至少达到：做过脚本、插件、自动化工作流、工具链封装之一

## 5. 大厂常见加分项

1. Agent 评测平台经验
- 如任务集、打分、回归测试、结果对比、轨迹回放

2. 轨迹可视化经验
- 如步骤回放、工具入参与出参查看、错误定位页面

3. LLM 系统认知
- 如 token 机制、context window、sampling、延迟与成本权衡

4. 开源贡献或技术输出
- 如 GitHub 项目、博客、演讲、社区回答

5. 安全或竞赛背景
- 如 CTF、沙箱、权限控制、工具隔离

## 6. 你最终应该做出的能力证明

想拿这类岗位，最好能拿出 3 个项目层面的证据：

1. Agent 主项目
- 例如：Agent 评测与轨迹平台
- 证明：你理解 Agent 链路、评测和平台化

2. 工程化项目
- 例如：容器化部署、CI/CD、日志监控、配置治理
- 证明：你不仅能写 Demo，还能做交付

3. 协议/底层项目
- 例如：MCP 接入层、Function Calling 适配层、Rust 工具运行时模块
- 证明：你具备一定底层与平台思维

## 7. 面试官真正会看什么

面试官通常不会只看“你学过没有”，而是看下面这些问题你能不能答出来：

1. 你做过什么真实项目？
2. 这个项目的核心链路是什么？
3. 为什么这么设计？有没有替代方案？
4. 线上失败了你怎么排查？
5. 如何评估效果提升？
6. 如何控制成本、延迟和稳定性？
7. 你用 AI coding 工具到底提升了什么？

## 8. 达标判断标准

如果你能同时做到下面这些，就基本接近可投递状态：

1. Python
- 能独立写 Agent 后端、评测脚本、测试和日志系统

2. TypeScript
- 能做一个前端可视化页面，并与后端联调

3. Rust
- 有一个小而完整的真实项目，而不是只学语法

4. Agent
- 能讲清 Tool Calling、MCP、Memory、RAG、评测体系

5. 平台能力
- 能把服务 Docker 化，理解 K8s 基本部署方式

6. 工程能力
- 有 Git、测试、CI/CD、代码审查习惯

7. 项目表达
- 能在 3 分钟内清楚讲完你的主项目架构、问题和结果

## 9. 对你最重要的现实建议

结合你当前背景，最优路径不是同时全面铺开，而是分层推进：

1. 第一阶段
- 先拿下 Python + TypeScript + FastAPI + React + Docker

2. 第二阶段
- 补 Agent 协议、评测平台、工具调用、轨迹可视化

3. 第三阶段
- 再补 Rust + Kubernetes + 稳定性治理

4. 最后阶段
- 打磨项目、简历、面试表达和 AI 编程工具工作流

## 10. 一句话版画像

目标岗位想要的人，不是“只会调模型 API 的应用开发”，而是：
一个既能写 Agent，又能把它做成稳定产品的人。

# Agent学习固化路线

更新时间：2026-04-26
定位：面向未来 1 年转型 Agent 全栈开发工程师的稳定学习主线
目的：只固化那些变化慢、长期有效的学习路线；每天会变的新概念、新框架、新工具，后续单独讨论更新机制

## 1. 总原则

这份路线只回答一件事：
什么东西值得你未来 1 年稳定投入，不会因为 AI 每周出新东西就被推翻。

核心结论：

1. 先学稳定能力，再追前沿变化
2. 先做出最小项目，再补平台深度
3. 先把主语言和工程能力打牢，再扩展到更复杂的 Agent 系统

整条路线分成 3 条稳定主线：

1. 基础概念主线
2. 语言与工程主线
3. AI 项目主线

它们分别对应你的时间结构：

1. 白天碎片时间：基础概念主线
2. 每晚 1 小时：语言与工程主线
3. 周末深度时间：AI 项目主线

## 2. 三条主线

### 2.1 基础概念主线

目标：
建立你对 LLM、Agent、工程系统的稳定认知框架。

这一条线不追求你当天写代码，而是追求：

1. 看得懂
2. 讲得出
3. 知道它和工程实现的关系

前期重点学这些稳定概念：

1. LLM 基础
- token
- context window
- prompt
- structured output
- function calling

2. Agent 基础
- workflow 和 agent 的区别
- 单 Agent / 多 Agent
- planning
- memory
- tool use
- multi-step execution

3. 检索与上下文
- RAG
- chunk
- retrieval
- rerank
- citation
- hallucination control

4. 基础工程概念
- HTTP / REST
- timeout / retry
- cache
- queue
- trace / log / metrics
- Docker / K8s / CI/CD

这一条线的达标标准：

1. 你能用自己的话解释核心概念
2. 你能把概念和一个工程场景联系起来
3. 你能把学到的概念映射到你的项目里

### 2.2 语言与工程主线

目标：
把“会看懂代码”升级成“能稳定写出工程代码”。

这一条线是你转型的硬地基，优先级最高。

学习顺序建议：

1. 第一阶段：Python
2. 第二阶段：TypeScript
3. 第三阶段：Go 或 Rust

前期不要同时把三门语言都拉满，先用 Python 打出第一个工程闭环。

这一条线的核心内容不是单纯语法，而是：

1. 语法与标准库
2. 项目结构
3. 包管理与虚拟环境
4. API 开发
5. 测试
6. 日志
7. 配置管理
8. lint / format
9. Docker 化

每门语言的目标不是“学完教程”，而是达到：

1. 能写功能
2. 能写测试
3. 能调试问题
4. 能按工程方式组织代码

### 2.3 AI 项目主线

目标：
把概念和语言能力装进真实项目，逐步形成可讲、可演示、可面试的作品。

这一条线是你未来求职最重要的证明材料。

项目路线建议从小到大演进：

1. 最小模型调用 Demo
2. 带 tool calling 的后端服务
3. 带前端展示的小系统
4. 带日志、测试、Docker 的完整项目
5. 带评测、轨迹、稳定性能力的 Agent 工程项目

这条线的关键原则：

1. 前期追求闭环，不追求复杂
2. 中期追求完整，不追求平台化
3. 后期追求稳定、评测和可扩展

## 3. 三阶段学习框架

## 第一阶段：前期（0-3个月）

目标：
先学会“做出来”。

这一阶段你最重要的事情只有 3 个：

1. 建立 Agent 和 LLM 的基础概念框架
2. 把 Python 和基本工程化能力学扎实
3. 做出最小 AI 项目闭环

这一阶段不需要过早引入太多复杂内容：

1. 不急着深挖 K8s
2. 不急着深挖 Rust
3. 不急着做完整评测平台
4. 不急着做复杂多 Agent 系统

前期建议达到的状态：

1. 你能写一个 Python API 服务
2. 你能接模型 API 和 tool calling
3. 你能写基础测试和日志
4. 你有一个最小可运行项目

前期代表性成果：

1. FastAPI 服务
2. 1 个 tool
3. 1 个模型调用链路
4. 1 份 README

## 第二阶段：中期（3-8个月）

目标：
从“做出来”升级到“做完整”。

这一阶段开始引入更多工程完整性：

1. TypeScript 前端
2. 前后端联调
3. Docker
4. 基础评测
5. 轨迹展示
6. 更清晰的错误处理和日志

这一阶段的重点不是追求底层深水区，而是把项目做得像一个像样的系统。

中期建议达到的状态：

1. 你能做一个简单前端页面
2. 你能让后端、前端、模型调用跑成闭环
3. 你能用 Docker 把项目跑起来
4. 你有基础评测和基本可观测能力

中期代表性成果：

1. 轨迹列表或详情页面
2. 批量评测脚本
3. Dockerfile
4. 项目 v1

## 第三阶段：后期（8-12个月）

目标：
从“做完整”升级到“做专业、做成大厂标准”。

这一阶段再补高阶内容最合适：

1. K8s
2. Rust 或 Go 的补强
3. 更完整的可观测性
4. 稳定性治理
5. 安全与权限控制
6. 系统设计与项目表达

后期建议达到的状态：

1. 你能讲清 Agent 系统架构和关键取舍
2. 你能解释线上问题怎么排查
3. 你能说明评测指标、成本、延迟、成功率之间的平衡
4. 你有 1 到 2 个能代表你能力的项目

后期代表性成果：

1. Agent 工程项目 v2
2. Docker + K8s 基础部署
3. Rust 或 Go 小模块
4. 面试项目讲解稿

## 4. 时间分配建议

为了让这条路线可长期执行，建议这样分配：

### 工作日白天碎片时间

只做基础概念主线：

1. 每次 10-20 分钟
2. 只看 1 个概念
3. 输出 3 句话笔记

目标不是学很多，而是持续建立认知。

### 工作日晚间 1 小时

只做语言与工程主线：

1. 10 分钟复习
2. 35 分钟编码
3. 15 分钟测试、重构、记录

目标不是刷知识点，而是稳定积累工程手感。

### 周末

优先做 AI 项目主线：

1. 做功能
2. 联调
3. 修问题
4. 补测试
5. 写文档

如果项目前期还没启动，周末也可以用一部分时间补语言与工程。

## 5. 学习顺序不要乱

你后面学习时，尽量不要犯这几个常见错误：

1. 只学概念，不做项目
2. 只学语法，不学工程化
3. 一开始就追多 Agent、K8s、Rust 深水区
4. 被每天的新框架和新名词带着跑

更稳的顺序应该是：

1. 先理解基础概念
2. 再把 Python 工程化学扎实
3. 再做最小 AI 项目
4. 再补前端、Docker、评测
5. 最后补平台深度与差异化能力

## 6. 什么叫达标

这份固化路线最终不是看你看了多少文章，而是看你是否达到这些状态：

1. 你能独立解释 Agent 基础概念
2. 你能独立写出一个像样的 Python Agent 后端
3. 你能做出至少一个可运行 AI 项目
4. 你能逐步把它补成完整系统
5. 你能在面试里清楚讲出项目架构、问题和取舍

## 7. 一句话总结

你这一年的稳定学习主线应该是：

白天学概念，晚上练语言和工程，周末做 AI 项目；先做出来，再做完整，最后做专业。

---

备注：这份文档故意不把“每天都在变化的 AI 新知识”写进主路线里，因为那部分应该作为动态更新层单独管理，不能干扰你的稳定能力建设。

# 每日阅读自动整理规则

这个文件用于约束后续自动新增阅读材料时的风格和范围。

目标：

每天为仓库补 1 到 2 篇高质量阅读材料，优先帮助当前阶段的 Agent 工程转型。

来源优先级：

1. 官方文档
2. 协议规范
3. 经典论文
4. 一线工程团队官方技术文章
5. 高质量框架官方文档

当前主题优先级：

1. Python 工程化
2. Tool Calling / Structured Outputs
3. MCP
4. Agent Workflow / State / Handoff
5. RAG
6. Eval / Trace / Observability
7. Spring AI / Java 桥接
8. Docker / K8s / 部署

新增材料格式要求：

1. 文件放在 `docs/`
2. 文件名按连续编号，例如 `13-xxx.md`
3. 必须包含：
- 原文来源
- 为什么今天读这个
- 原文翻译（先于中文精读）
- 中文精读
- 你只需要记住的 3 个点
- 和 Agent 工程的关系
- 今晚可动手
- 原文中文翻译链接（机器翻译）

质量要求：

1. 不要机械直译
2. 要结合“10 年 Java + IoT 集成 + ToB 交付 + 准备用 1 年转 Agent 工程”的背景来解释
3. 每篇文档控制在手机上可快速阅读的长度
4. 如果当天没有足够高质量的新材料，可以选择经典材料做更深入的精读，而不是凑数
5. 结构顺序必须固定：先“原文翻译”，再“中文精读”
6. 在“今晚可动手”中有“问自己两个问题”时，原文中文翻译链接必须紧跟在这两个问题后面

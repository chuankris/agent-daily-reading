# 每日行业雷达自动整理规则

目标：

每天新增 10 条行业动态（短摘要）+ 1 条重点解读，帮助学习者保持对 Agent 领域最新变化的判断力。

学习者背景：

1. 10 年 Java 后端与 IoT 系统集成经验
2. 正在转型 Agent 工程
3. 当前重点：Python、MCP、RAG、Eval、可观测、工程化落地

选题优先级：

1. 大厂/头部机构新模型发布（及其工程意义）
2. 关键技术路线分歧（例如 MoE、长上下文、Agent 编排范式、评测方法）
3. 新工具或平台能力更新（与开发效率、可运维性有关）
4. 高质量开源项目（可直接学习或复用）

质量门槛：

1. 优先官方来源、官方博客、官方文档、主仓库发布说明
2. 如果是媒体解读，必须有一手来源可追溯
3. 不做情绪化标题，不做纯流量热点

输出格式（每篇必须包含）：

1. 今日雷达主题
2. 事件摘要（发生了什么）
3. 原文要点翻译（人工翻译，简短）
4. 这件事为什么重要（技术与工程角度）
5. 对你当前路线的影响（只讲与你有关的）
6. 今晚可验证动作（10-20 分钟）
7. 原文链接（一手优先）
8. 原文中文翻译链接（机器翻译，补充）

控制原则：

1. 每天产出 10 条雷达条目，按相关度排序
2. 在 10 条中选 1 条做“重点解读”
3. 可读性优先，手机 8-12 分钟可读完
4. 强调“与你的转型相关”，避免泛新闻
